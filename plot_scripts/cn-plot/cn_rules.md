# 作图代码中文化规则

当用户要求将英文作图代码转换为中文作图代码时，遵循以下约定：

- 涉及中文的文本使用 `simsun.ttc`；其余文本使用 `Times_New_Roman.ttf`。
- 当同一段文本内同时包含中文、英文或数字时，优先使用 `simsun.ttc`，不额外复杂化字体拆分逻辑。
- 轴标题默认字号为 `10.5`。
- legend 默认字号为 `9.5`。
- 刻度数字默认字号为 `8.5`。
- 默认不添加图总标题。
- 尺寸输入单位为 `cm`，代码内部通过 `CM_TO_INCH` 转换为英寸。
- 中文化后的脚本文件名前缀使用 `cn_`。
- 其余设置项如无特别注明，且与上述规则不冲突，优先参照原英文图。

## 折线图 / Bar 图中文化通用约定

- 优先参考同目录下已有 `cn_*.py` 的风格，尤其是版式、配色、线宽、marker、legend 位置与坐标范围。
- 常用版式默认采用 `12 cm` 宽、`16:9`，并使用 `plt.tight_layout(pad=0.15)`。
- 中文轴标题、中文 legend 使用 `simsun.ttc`；刻度数字与纯数字标签使用 `Times_New_Roman.ttf`。
- bar 图默认保持原英文图的柱宽、颜色分组和误差棒逻辑；折线图默认保持原英文图的线型、marker 类型和数据顺序。
- 线条、柱边框、误差棒、坐标轴框线通常使用偏细风格；如无特别说明，优先与已有中文图一致，不额外夸张。
- 纵轴若原始数据是 `0~1` 比例，中文图通常转换为百分比显示，并将轴标题写成 `（%）` 形式。
- 图例与坐标轴标题翻译为中文，但方法名、模型名、专有缩写通常保留原文，除非用户明确要求翻译。
