## 2. 论文当前定义的目标系统

### 2.1 Sec II：任务与 MDP 定义

论文当前将系统定义为一个 **single-step in-contact grasp pose refinement** 问题。

其核心定义为：

- episode 语义：单次观测，单次 6-DoF corrective action，然后结束
- 状态：
  - `s = [ z || t ]`
  - `z`：由显式 3D 对齐的 visuo-tactile observation 提取出的 latent representation
  - `t`：低维、可解释的 contact semantics
- 动作：
  - `a = [ Δp, Δθ ]`
  - 即 3 维平移增量 + 3 维旋转增量
- 优化目标：
  - 学习随机策略 `π(a|s)`
  - 最大化 refinement 后的期望 reward

### 2.2 Sec III-A：感知状态

论文当前希望感知状态由两部分构成：

- 预训练 SGA-GSN backbone 提取的 visuo-tactile latent feature `z`
- 两个接触语义：
  - `t_cover`
  - `t_edge`

其论文语义是：

- tactile sensing points 构成集合 `P`
- valid contact point set 为 `C = { p_i in P | d_i > τ }`
- `t_cover = |C| / |P|`
- `t_edge` 表示 valid contact 到 sensor boundary 的归一化距离统计

### 2.3 Sec III-B：稳定性评估与在线校准

论文当前定义：

- 先由预训练稳定性预测器得到 raw stability logit：
  - `ŝ = g_φ(z)`
- 再通过两参数 logistic calibration 做在线调整：
  - `p_i = σ(β^T [ŝ_i, 1]^T)`
- 每个 rollout 后，利用 Bernoulli negative log-likelihood 更新 `β = [a, b]^T`
- Hessian 的逆用于近似 posterior covariance `P`
- 后续 reward 依赖：
  - calibrated stability
  - posterior uncertainty

### 2.4 Sec III-C：reward 与 PPO

论文当前 reward 由三项组成：

- `R_drop`
- `R_contact`
- `R_stability`

论文当前写法为：

- `R_drop`：
  - 成功为 `+1`
  - 失败为 `-1`
- `R_contact`：
  - 对 `t_cover` 和 `t_edge` 使用带阈值的 soft-hinge penalty
- `R_stability`：
  - 基于 `Δp = p(g1) - p(g0)`
  - 再由 `1 / (1 + κ tr(P))` 衰减

优化器采用 PPO actor-critic。

## 3. 当前工程实际已经落地的训练系统

当前仓库已经实际固定为：

- dataset-backed
- single-step
- grasp refinement RL

每个 episode 的实际语义是：

1. 从 tactile-extended dataset 采一个 sample
2. 用 sample 重建 pre-grasp 物理场景
3. 现场采集 live `before`
4. policy 输出一个 6-DoF normalized delta action
5. env 执行一次 refinement
6. 重建 refined grasp
7. 现场采集 live `after`
8. 执行一次 release trial
9. `done=True`

因此，当前系统确实已经是论文希望的单步 RL 形态，而不是旧的 heuristic pipeline。

## 4. 已完成且与论文基本一致的部分

本节表示：这些部分已经进入当前训练闭环，并且在系统层面与论文目标方向基本一致。

### 4.1 单步 grasp refinement RL 主形态

当前训练主环已经是：

- `reset -> observe before -> act -> refine -> observe after -> trial -> done`

这与论文的 single-step decision formulation 一致。

### 4.2 6-DoF pose delta action

当前 action 空间已经是：

- 3 维 translation delta
- 3 维 rotation delta

这与论文中的 `a = [ Δp, Δθ ]` 一致。

### 4.3 live before / live after / release trial

当前 `PyBulletScene` 已具备：

- 真实 hand URDF
- 真实 object URDF
- TACTO tactile sensor
- live `before`
- live `after`
- refine 后 release trial

当前成功/失败已经不再是旧的 target-pose heuristic，而是由真实仿真中的接触保持情况决定。

### 4.4 SGA-GSN backbone 路径已经进入正式训练链路

当前默认 perception 配置已经不是 placeholder backbone，而是：

- `adapter_type: sga_gsn`
- CPU 上构建 VTG3D 风格 `sc_input / gs_input`
- CUDA 上调用共享 `SGAGSNPerceptionRuntime`
- 使用 AdaPoinTr + SGSNet body + predict head

这意味着：

- 论文中的“预训练 visuo-tactile backbone 提取 latent feature”已经进入主训练链路
- 当前训练不是建立在 dummy latent 上

### 4.5 显式 3D 对齐输入重建主链已经落地

当前 adapter 已实现：

- visual `depth + seg + proj + view -> world points`
- tactile `depth + proj + gel pose -> gel/world points`
- tactile `step` 采样
- gel/contact mask
- zero-mean
- `sc_input`
- `gs_input`

因此，论文中“基于显式对齐 3D visuo-tactile observation 构造感知输入”的核心工程基础已经具备。

### 4.6 在线 calibrator 已接入训练闭环

当前系统已经有：

- 两参数 `a, b` calibrator
- `predict(logit) -> calibrated probability + uncertainty`
- 每轮 rollout 后，用执行结果更新 calibrator

因此，论文中的“online adjustment of stability prediction”已经进入真实训练闭环。

### 4.7 PPO actor-critic 已接入真实系统训练

当前训练入口已经能在真实配置下完成：

- rollout collection
- PPO update
- calibrator update

也就是说，从“能否支撑最终系统训练”角度，当前仓库已经具备 **真实单环境 end-to-end 训练能力**。

## 5. 已完成，但与论文当前写法不一致的部分

本节表示：这些能力已经接入当前工程并可用于训练，但与论文 Sec II / Sec III 当前写法并不严格一致。

### 5.1 policy 实际状态不是论文定义的 `s = [ z || t ]`

论文当前写法是：

- `s = [ z || t ]`

但当前 policy 实际输入为：

- `latent_feature`
- `contact_semantic`
- `grasp_pose.position`
- `grasp_pose.rotation`
- `raw_stability_logit`

即：

- 当前 policy 看到的不只是 `z + t`
- 还额外看到了 pose 和 raw logit

这意味着：

- 当前工程训练状态空间比论文定义更大
- 若论文最终坚持 `s = [ z || t ]`，则当前实现仍需收敛

### 5.2 reward 结构与论文定义不完全一致

当前代码仍然保留三项 reward 分解：

- drop
- contact
- stability

这一点与论文形式一致。

但具体公式不一致：

- 当前 `drop reward` 与论文一致，仍是成功 `+1` / 失败 `-1`
- 当前 `contact reward` 不是论文中的 soft-hinge threshold penalty
  - 现实现为 before/after contact semantic 增量形式
- 当前 `stability reward` 不是论文中的
  - `Δp / (1 + κ tr(P))`
  - 而是 `gain - alpha * positive_uncertainty_increase`

因此，当前 reward 是“论文意图的工程近似版”，不是论文公式的严格实现。

### 5.3 calibrator 更新形式与论文公式不完全一致

论文当前更接近：

- 每个 rollout 用一次 Newton / IRLS 风格更新
- `P ≈ (H + λI)^(-1)`
- reward 中使用 `tr(P)`

当前实现是：

- logistic calibrator
- 带 `learning_rate`
- 带 `l2_reg`
- 带 `prior_var`
- 输出的是对单个 logit 的预测不确定度

因此：

- 方向上与论文一致
- 数学形式上不是论文当前写法的严格复现

### 5.4 当前 perception 文档与当前代码状态不一致

`v3.md` 仍把 perception 描述成包含明显 placeholder 的状态，例如：

- point cloud 由 adapter dummy 占位
- SGA-GSN encoder 仍是 placeholder

但当前代码已经不是这个状态：

- visual/tactile world reconstruction 已存在
- `sc_input / gs_input` 已存在
- 共享 `SGAGSNPerceptionRuntime` 已存在
- 真实 checkpoint 加载与前向也已可运行

因此：

- `v3.md` 在 perception 部分已经滞后
- 本文件应视为更新后的对齐基线

## 6. 仍未完成的部分

本节表示：这些点若不补齐，则当前工程仍不能被视为“论文当前系统定义的完整实现”。

### 6.1 contact semantics 仍是 placeholder，不是论文公式

这是当前最关键的未完成项之一。

论文期望的 `t_cover / t_edge` 是：

- 由 tactile geometry 中的 valid contact set `C` 推导
- 明确对应 contact coverage 与 boundary proximity

但当前 `ContactSemanticsExtractor` 仍然只是：

- 基于 tactile signal 阈值统计 active ratio
- 基于左右 sensor 激活差异构造 edge-like 指标

这说明：

- 当前 contact semantics 可以训练
- 但它不是论文定义的那两个量
- 论文 Sec III-A 尚未在工程上闭合

### 6.2 reward 尚未按论文公式收敛

若论文最终保持当前写法，则至少还需补齐：

- `R_contact` 的 thresholded soft-hinge 形式
- `R_stability = Δp / (1 + κ tr(P))`

当前 reward 虽然可训练，但仍不是论文定义版。

### 6.3 policy observation 尚未收敛到论文定义

若论文最终坚持状态仅为 `s = [ z || t ]`，则当前还未完成：

- 去掉额外拼入 policy 的 `grasp_pose`
- 去掉额外拼入 policy 的 `raw_stability_logit`
- 重新确认 actor / critic 的 observation contract

### 6.4 多环境正式训练能力尚未完成

当前真实可确认的是：

- 单环境真实训练可运行

但当前仍不能把“多环境正式训练”视为已完成，原因是：

- 当前 `DummyVecEnvWrapper` 只是同步包装，不是真并行 runtime
- `PyBulletScene` 当前默认假设单进程单 Bullet client
- 当前规划中的正式并行方式仍应是多进程，而非单进程多 world

因此：

- `num_envs > 1` 目前不能被视为论文系统训练能力的正式完成项

### 6.5 论文文本本身仍有草稿痕迹

当前论文源码仍存在：

- `abstract` 占位
- `experiments` 占位
- `conclusions` 占位
- Sec II / Sec III 中若干黄色批注与待清理语句

这不影响工程训练本身，但意味着：

- 当前论文定义仍是工作稿
- 本文件只能对齐“当前草稿所表达的系统目标”，而不是最终定稿版本

## 7. 从“最终支持系统训练”角度的结论

### 7.1 当前已经可以确认支持的能力

当前仓库已经可以确认支持：

- 真实数据集采样
- 真实 scene reset / refine / release
- live `before / after`
- 真实 SGA-GSN perception runtime
- online calibrator
- PPO actor-critic
- 单环境 end-to-end 训练闭环

因此，当前系统已经不是“只能做模块级调试”，而是已经具备 **真实训练原型**。

### 7.2 当前尚不能宣称完全实现论文系统

当前仍不能宣称“论文 Sec II / Sec III 已完整实现”，主要因为：

- contact semantics 仍不是论文定义版
- reward 仍不是论文公式版
- policy state 仍不是论文定义版

所以更准确的表述应为：

> 当前工程已经完成论文系统的训练主干与核心 runtime 闭环，但在状态定义、接触语义和 reward 公式上仍处于“可训练的工程近似版”，尚未完全收敛到论文当前写法。
