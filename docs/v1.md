# 1. 项目目标与冻结假设

## 1.1 任务定义

本项目实现一个 **single-step in-contact grasp pose refinement** 系统。

给定：

* 初始抓取状态 `initial grasp`
* 当前接触后的观测
* 由显式对齐 3D 视触觉表征提取的 latent feature
* 可解释接触语义
* 初始 grasp 的 stability raw logit

策略输出：

* 一个小幅 6-DoF 修正动作

环境执行：

* 对初始 grasp 做 refinement
* 运行 grasp / hold / drop 测试
* 返回修正后的 outcome 与 reward

---

## 1.2 reward 定义

第一版 reward 由三部分组成：

* `R_drop`: refined grasp 的最终成败硬反馈
* `R_stability`: calibrated stability after-before improvement
* `R_contact`: 接触语义先验项

其中：

* `before` 主要是参考状态，不做单独真实物理成功率测试
* `after` 才做真实执行检验

---

## 1.3 已冻结的系统边界

* 单步 episode，`done=True` after one action
* env 与 RL trainer 分离
* policy/value network 与 trainer 分离
* perception 负责感知逻辑封装，不直接定义网络
* 网络结构定义放到 `models/`
* calibrator 独立成类，不绑死 predictor
* uncertainty 作为 calibrator 内部能力，不单独拆第一版模块
* I/O 尽量结构化，不急着全拍扁成 tensor

## `reset()` 应做的事

1. sample 场景
2. scene reset
3. 设置 initial grasp
4. 采 raw obs
5. build `obs_before`
6. 缓存 `obs_before` / `grasp_pose_before`

## `step()` 应做的事

1. decode action
2. 得到 refined pose
3. scene 执行 refined pose
4. scene 跑真实 trial
5. 采 after raw obs
6. build `obs_after`
7. calibrator 对 before/after logits 做预测
8. reward manager 计算 reward
9. done = True
10. 返回 next_obs / scalar reward / done / info

## 哪些必须先写真逻辑

* `reset`
* `step`

## 哪些可先 stub

* `_sample_initial_state` 可以先简单随机
* `_get_initial_grasp_pose` 可以先从 sample cfg 直接读

# 13. 一次 rollout 的标准调用链

这是给 codex 最关键的“串联系统图”。

## 13.1 reset 阶段

```text
Trainer.collect_rollout()
    -> env.reset()
        -> env._sample_initial_state()
        -> scene.reset_scene(sample_cfg)
        -> env._get_initial_grasp_pose(sample_cfg)
        -> scene.set_initial_grasp(grasp_pose_before)
        -> scene.get_raw_observation()
        -> observation_builder.build(raw_obs_before, grasp_pose_before)
            -> feature_extractor.extract(raw_obs_before)
                -> adapter.adapt_feature_input(raw_obs_before)
                -> backbone_model.forward(...)
            -> contact_semantics_extractor.extract(raw_obs_before)
            -> stability_predictor.predict_logit(raw_obs_before, latent_feature)
                -> adapter.adapt_predictor_input(...)
                -> predictor_model.forward(...)
        -> return obs_before
```

---

## 13.2 action 采样阶段

```text
Trainer.collect_rollout()
    -> observation_to_tensor(obs_before)
    -> actor_critic.act(obs_tensor)
        -> policy_net.forward(...)
        -> sample Normal distribution
        -> value_net.forward(...)
    -> 得到 action, log_prob, value
```

---

## 13.3 step 阶段

```text
Trainer.collect_rollout()
    -> env.step(action)
        -> action_executor.decode(action)
        -> action_executor.apply_to_pose(grasp_pose_before, physical_action)
        -> scene.apply_refinement(grasp_pose_after)
        -> scene.run_grasp_trial()
        -> scene.get_raw_observation()
        -> observation_builder.build(raw_obs_after, grasp_pose_after)
        -> calibrator.predict(raw_logit_before)
        -> calibrator.predict(raw_logit_after)
        -> reward_manager.compute(...)
        -> termination.is_done() -> True
        -> env._build_step_info(...)
        -> return obs_after, reward.total, True, info
```

---

## 13.4 rollout 存储与更新阶段

```text
Trainer.collect_rollout()
    -> buffer.add(...)

Trainer.train()
    -> batch = buffer.get_all()
    -> agent.update(batch)
    -> trainer.update_calibrator()
        -> buffer.get_all()
        -> extract raw_logit_after + drop_success
        -> calibrator.update(logits, labels)
    -> logger.log_dict(...)
    -> buffer.clear()
```

# 16. 给 codex 的实现约束

下面这几条建议你原样交给 codex，避免它写歪：

1. 所有模块之间优先通过结构化 dataclass 传递数据，不要过早拍扁。
2. `models/` 只定义网络结构，不写训练循环。
3. `perception/` 负责实例化和适配，不直接承担 RL 逻辑。
4. `envs/` 不得感知 PPO、advantage、optimizer 等训练细节。
5. calibrator 必须是独立类，且支持 rollout-level `update()`。
6. `RewardManager` 必须返回分项 reward，而不是只返回 total scalar。
7. 第一版只实现 single-step episode。
8. 第一版只对 refined grasp 做真实 trial outcome，不对 initial grasp 做单独真实试验。
9. policy/value 使用最简单可运行 MLP，不要提前过度复杂化。
10. 所有关键模块都应带最基本 docstring 和 type hints。
